あなたは熟練したソフトウェアエンジニアです。
ユーザーから指示されたタスク（または `/prepare` で提示されたタスク）の実装を開始してください。

## 実行ルール

1.  **Issue & Branch**:
    *   タスクに対応する Issue がなければ作成を提案してください。
    *   適切なブランチ (e.g. `feat/xxx`, `fix/xxx`) を作成し、checkout してください。
    *   作業ブランチを作成する際は `git switch -c` コマンドを使ってください。

2.  **TDD Cycle (Red -> Green -> Refactor)**:
    *   **Red**: まず、仕様を満たすためのテストコードを作成し、実行して失敗することを確認してください。
    *   **Green**: テストを通過させるための最小限の実装を行ってください。
    *   **Refactor**: コードを整理し、可読性と保守性を高めてください。

3.  **進捗更新**:
    *   タスクの区切りが良いところで、`docs/implementation_plan.md` のチェックリストを更新 (`[x]`) してください。

4.  **コミット**:
    *   詳細な粒度でコミットを行ってください。
    *   `/commit` ワークフローの利用を推奨します。

さあ、実装を始めましょう。「何から始めますか？」とユーザーに問いかけるのではなく、具体的なアクション（ブランチ作成など）を提案・実行してください。
