---
description: 変更内容を解析して git commit を行う
---

1. **状況確認とステージング**:
   - `git status` を確認してください。
   - ステージングされた変更がない場合は、`git add .` を実行して全ての変更をステージングしてください。
   - `git diff --cached` を実行して、変更内容を詳細に分析してください。

2. **コミットメッセージの生成**:
   - 変更内容に基づき、[Conventional Commits](https://www.conventionalcommits.org/ja/v1.0.0/) 形式でコミットメッセージを作成してください。
   - **フォーマット**: `type(scope): subject`
   - **言語**: 日本語
   - **Type一覧**:
     - `feat`: 新機能の追加
     - `fix`: バグ修正
     - `docs`: ドキュメントの変更
     - `style`: フォーマットの変更（コードの動作に影響しない）
     - `refactor`: バグ修正や機能追加を含まないコードの変更
     - `perf`: パフォーマンスの向上
     - `test`: テストの追加や修正
     - `chore`: ビルド設定やツールの変更
   - **Subject**: 変更内容を簡潔かつ具体的に要約してください。

3. **コミットの実行**:
   - `git commit -m "生成したメッセージ"` を実行してください。
