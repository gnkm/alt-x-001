# Product Roadmap

## Phase 1: MVP（最小限の機能で動作する状態）

1. [ ] ユーザー認証（ログイン） — ユーザー登録・ログイン・ログアウト機能を実装。JWT認証により、認証済みユーザーのみがアプリにアクセスできる状態にする。 `M`

2. [ ] 基本投稿機能 — テキストコンテンツを投稿できる機能を実装。投稿はデータベースに保存され、作成日時が記録される。 `S`

3. [ ] 投稿一覧表示 — 保存された投稿を一覧で表示する機能を実装。新しい投稿が上に表示される時系列順で表示する。 `S`

4. [ ] 投稿削除 — 一覧画面から任意の投稿を削除できる機能を実装。削除確認ダイアログを表示し、誤削除を防止する。 `XS`

## Phase 2: コア機能の強化

5. [ ] タグ付け機能 — 投稿時にタグを追加できる機能を実装。複数タグの設定、タグによるフィルタリングが可能。 `M`

6. [ ] 投稿検索機能 — キーワードで投稿を検索できる機能を実装。投稿本文とタグを対象に部分一致検索を行う。 `S`

7. [ ] 画像ファイル添付 — 投稿に画像ファイルを添付できる機能を実装。アップロードされた画像はローカルストレージに保存され、投稿と一緒に表示される。 `M`

## Phase 3: パフォーマンス最適化

8. [ ] 遅延ロード（Infinite Scroll） — 投稿一覧の遅延ロードを実装。スクロールに応じて追加の投稿を読み込み、大量の投稿でもスムーズに閲覧できる。 `S`

---

> Notes
> - Order items by technical dependencies and product architecture
> - Each item should represent an end-to-end (frontend + backend) functional and testable feature
> - Phase 1 完了時点で、基本的な投稿・閲覧・削除が可能な MVP が動作する
> - Phase 2 で検索性・整理機能を強化
> - Phase 3 でユーザー体験を向上させるパフォーマンス最適化を実施
